# 出库数量同步至销售订单功能
- 当新增出库单选择了销售订单并保存后，把出库单中每样商品出库的数量同步至销售订单sales-items属性的的outbound_quantity属性中
- 同步方式为：sales-items中的出库数量与出库单中的出库数量相加
- 数据处理：如果相加之后的值刚好等于销售订单中该商品的数量，则把该商品的状态改为已出库，如果大于销售订单中该商品的数量，则在出库单出库数量处报错：已超出销售数量，如果保存时销售订单该商品的出库数量为0或者空，则把该商品的状态从未出库改为已部分出库
# 入库数量同步至采购订单功能
- 当新增入库单选择了采购订单并保存后，把入库单中每样商品入库的数量同步至采购订单purchase-items属性的的inbound_quantity属性中
- 同步方式为：purchase-items中的入库数量与入库单中的入库数量相加
- 数据处理：如果相加之后的值刚好等于采购订单中该商品的数量，则把该商品的状态改为已入库，如果大于采购订单中该商品的数量，则在入库单入库数量处报错：已超出采购数量，如果保存时采购订单该商品的入库数量为0或者空，则把该商品的状态从未入库改为已部分入库

## 注意
销售订单的sales-items属性时一个json字符串，里面已经存在出库数量outbound_quantity，无需再添加outbound_quantity属性，采购订单同理
